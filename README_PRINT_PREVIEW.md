## 🔍 Feature Overview

### What It Does
Shows a preview window of what the thermal printer receipt will look like before actually printing.

### Who Uses It
- Anyone with a thermal printer connected
- Anyone who wants to verify ticket format
- Users setting up their printer for the first time

### Key Benefits
✅ See format before wasting paper
✅ No printer required for testing
✅ Customizable header and width
✅ Copy text for documentation
✅ Automatic printing during actual transactions

## 💡 Pro Tips

### Tip 1: Test First
Always click 🖨️ Test Print before enabling auto-print to verify format.

### Tip 2: Customize Header
Edit the Header field in Print Ticket settings to match your company info.

### Tip 3: Match Paper Width
Set paper width to match your actual printer (usually 32 characters).

### Tip 4: Copy for Records
Use 📋 Copy Text to save ticket examples for documentation.

### Tip 5: Enable Auto-Print Last
Configure everything first, then check "Auto-print after capture" to enable.

## ❓ Frequently Asked Questions

**Q: Do I need a printer to use the preview?**
A: No! Preview works without any printer connected.

**Q: Can I edit the preview text?**
A: No, but you can copy it and edit in a text editor, or customize settings.

**Q: What if my printer isn't listed?**
A: Check COM port selection and verify printer is connected.

**Q: Can I change the ticket format?**
A: Yes! Edit header text and paper width in Print Ticket settings.

**Q: What if print fails?**
A: Check printer connection, COM port, and baud rate settings.

See [PRINT_PREVIEW_FEATURE.md](PRINT_PREVIEW_FEATURE.md) for more FAQs.
