                    PRINT PREVIEW FEATURE - QUICK SUMMARY

WHAT WAS CREATED:
  A complete print preview system for the thermal printer ticket functionality.
  Users can now see exactly what their receipt will look like before printing.

HOW TO USE:
  1. Click 🖨️ Test Print button in Print Ticket settings panel
  2. Preview window opens showing sample ticket format
  3. Review formatting, customize if needed
  4. Click 🖨️ Print Ticket to send to printer, or 📋 Copy Text to copy to 
     clipboard, or Close to dismiss

KEY FEATURES:
  ✅ Preview window shows exact ticket format
  ✅ No printer connection required for testing
  ✅ Customizable header text (company name, address, etc.)
  ✅ Adjustable paper width (24-48 characters)
  ✅ Copy ticket text to clipboard
  ✅ Automatic printing for real transactions
  ✅ Professional error handling and validation
  ✅ Fully backward compatible with existing system

TICKET SHOWS:
  • Batch ID
  • Date and Time
  • Ingredient Name
  • Target Weight
  • Actual Weight
  • Tolerance
  • Status (UNDER/OK/OVER)
  • Deviation from Target

TICKET FORMAT EXAMPLE:
  ================================
           COMPANY NAME
           Addr • Tel
  ================================

  Batch: TEST-001
  Time:  01/18/2026 02:30:45 PM

  Item:  Test Ingredient

  Target: 1000.00 g
  Actual: 1005.30 g
  Tol:    2.0%

  Status: OK
  Dev:    +5.30 g

  ================================

CONFIGURATION OPTIONS:
  • Header Text: Customize company name/address
  • Paper Width: Set to printer width (24-48 chars, default 32)
  • COM Port: Select printer serial port
  • Baud Rate: Set communication speed (typical 9600)
  • Parity: None/Even/Odd (typically None)

BUTTON FUNCTIONS:
  🖨️ Print Ticket  → Send ticket to printer (if connected)
  📋 Copy Text     → Copy ticket text to clipboard
  Close            → Close preview window

AUTO-PRINT MODE:
  When "Auto-print after capture" is enabled:
  • Each captured weight automatically sends to printer
  • Uses actual batch ID and ingredient names
  • Prints in background (non-blocking)
  • User receives confirmation of print status
